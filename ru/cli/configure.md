# `openclaw configure`

Интерактивный запрос для целевых изменений существующей настройки: учетные данные, устройства, значения по умолчанию для агентов, Gateway, каналы, плагины, Skills и проверки работоспособности.

Используйте `openclaw onboard` для полного пошагового первого запуска, `openclaw setup` только для базовой конфигурации/рабочей области, а `openclaw channels add`, когда нужна только настройка учетной записи канала.

Когда configure запускается с выбора аутентификации провайдера, средства выбора модели по умолчанию и списка разрешенных автоматически отдают предпочтение этому провайдеру. Для парных провайдеров, таких как Volcengine и BytePlus, то же предпочтение также соответствует их вариантам плана для программирования (`volcengine-plan/*`, `byteplus-plan/*`). Если фильтр предпочитаемого провайдера дал бы пустой список, configure возвращается к нефильтрованному каталогу вместо того, чтобы показывать пустой выбор.

Для веб-поиска `openclaw configure --section web` позволяет выбрать провайдера и настроить его учетные данные. Некоторые провайдеры также показывают дополнительные последующие запросы, зависящие от провайдера:

  * **Grok** может предложить необязательную настройку `x_search` с тем же профилем xAI OAuth или API-ключом и позволить выбрать модель `x_search`.
  * **Kimi** может запросить регион Moonshot API (`api.moonshot.ai` или `api.moonshot.cn`) и модель веб-поиска Kimi по умолчанию.

## Параметры

  * `--section <section>`: повторяемый фильтр разделов


Доступные разделы:

  * `workspace`
  * `model`
  * `web`
  * `gateway`
  * `daemon`
  * `channels`
  * `plugins`
  * `skills`
  * `health`


Примечания:

  * Полный мастер и разделы, связанные с Gateway, спрашивают, где работает Gateway, и обновляют `gateway.mode`. Фильтры разделов, которые не включают `gateway`, `daemon` или `health`, переходят непосредственно к запрошенной настройке.
  * После записи локальной конфигурации configure устанавливает выбранные загружаемые плагины, когда выбранный путь настройки требует их. Конфигурация удаленного Gateway не устанавливает локальные пакеты плагинов.
  * Сервисы, ориентированные на каналы (Slack/Discord/Matrix/Microsoft Teams), во время настройки запрашивают списки разрешенных каналов/комнат. Можно вводить имена или ID; мастер по возможности преобразует имена в ID.
  * Если вы запускаете шаг установки демона, аутентификация по токену требует токен, а `gateway.auth.token` управляется через SecretRef, configure проверяет SecretRef, но не сохраняет разрешенные значения токена в открытом тексте в метаданные окружения supervisor-сервиса.
  * Если аутентификация по токену требует токен, а настроенный SecretRef токена не разрешен, configure блокирует установку демона с практическими указаниями по исправлению.
  * Если настроены и `gateway.auth.token`, и `gateway.auth.password`, а `gateway.auth.mode` не задан, configure блокирует установку демона, пока режим не будет задан явно.

يرسل `/steer` إرشادات إلى تشغيل نشط بالفعل. يُستخدم للحظات "عدّل هذا التشغيل بينما لا يزال يعمل"، وليس لبدء دور جديد.

## الجلسة الحالية

استخدم `/steer` على المستوى الأعلى لاستهداف التشغيل النشط للجلسة الحالية:

textCopy code
[code]
    /steer prefer the smaller patch and keep the tests focused/tell summarize before making the next tool call
[/code]

السلوك:

  * يستهدف التشغيل النشط للجلسة الحالية فقط.
  * يعمل بشكل مستقل عن وضع `/queue` الخاص بالجلسة.
  * لا يبدأ تشغيلاً جديداً عندما تكون الجلسة خاملة.
  * يرد بتحذير عندما لا يوجد تشغيل نشط لتوجيهه.
  * يستخدم مسار التوجيه الخاص بوقت التشغيل النشط، لذلك يرى النموذج الإرشادات عند حد وقت التشغيل المدعوم التالي.


## التوجيه مقابل قائمة الانتظار

يغيّر `/queue steer` كيفية تصرف الرسائل الواردة العادية عندما تصل بينما يكون تشغيل نشطاً. أما `/steer <message>` فهو أمر صريح يحاول حقن رسالة ذلك الأمر في التشغيل النشط عند حد وقت التشغيل المدعوم التالي، بغض النظر عن إعداد `/queue` المخزّن.

استخدم:

  * `/steer <message>` عندما تريد توجيه التشغيل النشط الآن.
  * `/queue steer` عندما تريد أن توجّه الرسائل العادية المستقبلية عمليات التشغيل النشطة افتراضياً.
  * `/queue collect` أو `/queue followup` عندما ينبغي للرسائل الجديدة أن تنتظر دوراً لاحقاً بدلاً من توجيه التشغيل النشط.


لأوضاع قائمة الانتظار وسلوك الرجوع الاحتياطي، راجع [قائمة انتظار الأوامر](</ar/concepts/queue>) و [قائمة انتظار التوجيه](</ar/concepts/queue-steering>).

## الوكلاء الفرعيون

استخدم `/subagents steer` عندما يكون الهدف تشغيلاً فرعياً:

textCopy code
[code]
    /subagents steer 2 focus only on the API surface
[/code]

لا يختار `/steer` على المستوى الأعلى وكيلاً فرعياً حسب المعرّف أو فهرس القائمة. فهو دائماً يستهدف التشغيل النشط للجلسة الحالية. راجع [الوكلاء الفرعيون](</ar/tools/subagents>) للاطلاع على معرّفات الوكلاء الفرعيين وتسمياتهم وأوامر التحكم.

## جلسات ACP

استخدم `/acp steer` عندما يكون الهدف جلسة إطار ACP:

textCopy code
[code]
    /acp steer --session agent:main:acp:codex tighten the repro
[/code]

راجع [وكلاء ACP](</ar/tools/acp-agents>) للاطلاع على اختيار جلسة ACP وسلوك وقت التشغيل.

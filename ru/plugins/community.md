CapabilitiesPlugins

Community plugins — это сторонние пакеты, которые расширяют OpenClaw каналами, инструментами, провайдерами, хуками или другими возможностями. Используйте [ClawHub](</ru/clawhub>) как основную поверхность обнаружения публичных Community plugins.

## Найти Plugin

Ищите в ClawHub из CLI:

bashCopy code
[code]
    openclaw plugins search "calendar"
[/code]

Установите Plugin из ClawHub с явным префиксом источника:

bashCopy code
[code]
    openclaw plugins install clawhub:<package-name>
[/code]

npm остается поддерживаемым путем прямой установки на время перехода запуска:

bashCopy code
[code]
    openclaw plugins install npm:<package-name>
[/code]

Используйте [Управление Plugin](</ru/plugins/manage-plugins>) для распространенных примеров установки, обновления, проверки и удаления. Используйте [`openclaw plugins`](</ru/cli/plugins>) для полного справочника команд и правил выбора источника.

## Публикация Plugin

Публикуйте публичные Community plugins в ClawHub, если хотите, чтобы пользователи OpenClaw могли находить и устанавливать их. ClawHub владеет актуальным списком пакетов, историей релизов, статусом сканирования и подсказками по установке; документация не ведет статический каталог сторонних Plugin.

bashCopy code
[code]
    clawhub package publish your-org/your-plugin --dry-runclawhub package publish your-org/your-plugin
[/code]

Перед публикацией убедитесь, что у Plugin есть метаданные пакета, манифест Plugin, документация по настройке и ясно обозначенный ответственный за сопровождение. ClawHub проверяет область владельца, имя пакета, версию, ограничения файлов и метаданные источника, прежде чем создать релиз, а затем скрывает новые релизы от обычных поверхностей установки и скачивания до завершения проверки и верификации.

Используйте этот контрольный список перед публикацией:

Требование | Зачем  
---|---  
Опубликован в ClawHub | Пользователям нужны рабочие подсказки `openclaw plugins install`  
Публичный репозиторий GitHub | Проверка исходного кода, отслеживание задач, прозрачность  
Документация по настройке и использованию | Пользователям нужно знать, как его настроить  
Активное сопровождение | Недавние обновления или оперативная обработка задач  
  
Используйте эти страницы для полного контракта публикации:

  * [Публикация ClawHub](</ru/clawhub/publishing>) объясняет владельцев, области, релизы, ревью, проверку пакетов и передачу пакетов.
  * [Создание Plugin](</ru/plugins/building-plugins>) показывает форму пакета Plugin и рабочий процесс первой публикации.
  * [Манифест Plugin](</ru/plugins/manifest>) определяет поля нативного манифеста Plugin.

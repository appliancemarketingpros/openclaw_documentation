## تسجيل الدخول اليدوي (موصى به)

عندما يتطلب موقع ما تسجيل الدخول، **سجّل الدخول يدويًا** في ملف تعريف متصفح **المضيف** (متصفح openclaw).

**لا** تعطِ النموذج بيانات اعتمادك. غالبًا ما تؤدي عمليات تسجيل الدخول الآلية إلى تفعيل دفاعات مكافحة الروبوتات وقد تتسبب في قفل الحساب.

العودة إلى وثائق المتصفح الرئيسية: [المتصفح](</ar/tools/browser>).

## أي ملف تعريف Chrome يُستخدم؟

يتحكم OpenClaw في **ملف تعريف Chrome مخصص** (اسمه `openclaw`، وواجهة مستخدم مائلة إلى البرتقالي). هذا منفصل عن ملف تعريف متصفحك اليومي.

بالنسبة إلى استدعاءات أداة المتصفح الخاصة بالوكيل:

  * الخيار الافتراضي: ينبغي للوكيل استخدام متصفح `openclaw` المعزول الخاص به.
  * استخدم `profile="user"` فقط عندما تكون جلسات تسجيل الدخول الحالية مهمة ويكون المستخدم حاضرًا أمام الحاسوب للنقر/الموافقة على أي مطالبة إرفاق.
  * إذا كانت لديك ملفات تعريف متعددة لمتصفح المستخدم، فحدّد ملف التعريف صراحةً بدلًا من التخمين.


طريقتان سهلتان للوصول إليه:

  1. **اطلب من الوكيل فتح المتصفح** ثم سجّل الدخول بنفسك.
  2. **افتحه عبر CLI** :

bashCopy code
[code]
    openclaw browser startopenclaw browser open https://x.com
[/code]

إذا كانت لديك ملفات تعريف متعددة، فمرّر `--browser-profile <name>` (الافتراضي هو `openclaw`).

## X/Twitter: التدفق الموصى به

  * **القراءة/البحث/المحادثات المتسلسلة:** استخدم متصفح **المضيف** (تسجيل دخول يدوي).
  * **نشر التحديثات:** استخدم متصفح **المضيف** (تسجيل دخول يدوي).


## العزل + الوصول إلى متصفح المضيف

جلسات المتصفح المعزولة **أكثر عرضة** لتفعيل اكتشاف الروبوتات. بالنسبة إلى X/Twitter (والمواقع الصارمة الأخرى)، فضّل متصفح **المضيف**.

إذا كان الوكيل معزولًا، فستستخدم أداة المتصفح العزل افتراضيًا. للسماح بالتحكم في المضيف:

json5Copy code
[code]
    {  agents: {    defaults: {      sandbox: {        mode: "non-main",        browser: {          allowHostControl: true,        },      },    },  },}
[/code]

ثم افتح متصفح المضيف بنفسك (استدعاءات CLI تعمل دائمًا على متصفح المضيف):

bashCopy code
[code]
    openclaw browser open https://x.com --browser-profile openclaw
[/code]

يمكن عندها لاستدعاءات أداة `browser` الخاصة بالوكيل استهداف المضيف بمجرد تعيين `sandbox.browser.allowHostControl: true`. بدلًا من ذلك، عطّل العزل للوكيل الذي ينشر التحديثات.

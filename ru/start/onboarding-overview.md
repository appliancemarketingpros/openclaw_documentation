Get startedFirst steps

OpenClaw предлагает два пути первичной настройки. Оба настраивают аутентификацию, Gateway и необязательные каналы чатов — они отличаются только тем, как вы взаимодействуете с настройкой.

## Какой путь выбрать?

| Первичная настройка через CLI | Первичная настройка в приложении macOS  
---|---|---  
**Платформы** | macOS, Linux, Windows (нативно или WSL2) | Только macOS  
**Интерфейс** | Мастер в терминале | Пошаговый интерфейс в приложении  
**Лучше для** | Серверов, сред без графического интерфейса, полного контроля | Настольного Mac, визуальной настройки  
**Автоматизация** | `--non-interactive` для скриптов | Только вручную  
**Команда** | `openclaw onboard` | Запустите приложение  
  
Большинству пользователей стоит начать с **первичной настройки через CLI** — она работает везде и дает максимальный контроль.

## Что настраивает первичная настройка

Независимо от выбранного пути первичная настройка задает:

  1. **Провайдер модели и аутентификация** — API-ключ, OAuth или токен настройки для выбранного провайдера
  2. **Рабочее пространство** — каталог для файлов агентов, шаблонов начальной загрузки и памяти
  3. **Gateway** — порт, адрес привязки, режим аутентификации
  4. **Каналы** (необязательно) — встроенные и поставляемые в комплекте каналы чатов, такие как iMessage, Discord, Feishu, Google Chat, Mattermost, Microsoft Teams, Telegram, WhatsApp и другие
  5. **Демон** (необязательно) — фоновая служба, чтобы Gateway запускался автоматически


## Первичная настройка через CLI

Запустите в любом терминале:

bashCopy code
[code]
    openclaw onboard
[/code]

Добавьте `--install-daemon`, чтобы также установить фоновую службу за один шаг.

Полный справочник: [Первичная настройка (CLI)](</ru/start/wizard>) Документация по команде CLI: [`openclaw onboard`](</ru/cli/onboard>)

## Первичная настройка в приложении macOS

Откройте приложение OpenClaw. Мастер первого запуска проведет вас через те же шаги в визуальном интерфейсе.

Полный справочник: [Первичная настройка (приложение macOS)](</ru/start/onboarding>)

## Пользовательские или отсутствующие в списке провайдеры

Если вашего провайдера нет в списке первичной настройки, выберите **Пользовательский провайдер** и введите:

  * Режим совместимости API (совместимый с OpenAI, совместимый с Anthropic или автоопределение)
  * Базовый URL и API-ключ
  * ID модели и необязательный псевдоним


Несколько пользовательских конечных точек могут сосуществовать — каждая получает собственный ID конечной точки.

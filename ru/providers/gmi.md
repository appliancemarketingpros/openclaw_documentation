ModelsProviders

GMI Cloud — это размещенная платформа инференса для передовых моделей и моделей с открытыми весами за OpenAI-совместимым API. В OpenClaw это официальный внешний поставщик Plugin, что означает: вы устанавливаете его один раз, выбираете его по идентификатору поставщика `gmi`, сохраняете учетные данные через обычную аутентификацию моделей и используете ссылки на модели вроде `gmi/google/gemini-3.1-flash-lite`.

Используйте GMI, когда вам нужен один API-ключ для нескольких семейств размещенных моделей, включая маршруты Google, Anthropic, OpenAI, DeepSeek, Moonshot и Z.AI, представленные в каталоге GMI. Он полезен как вторичный поставщик для резервного выбора моделей, для сравнения размещенных маршрутов разных поставщиков или когда модель становится доступна в GMI раньше, чем у вашего основного поставщика.

Этот поставщик использует OpenAI-совместимую семантику чата. OpenClaw отвечает за идентификатор поставщика, профиль аутентификации, псевдонимы, начальный каталог моделей и базовый URL; GMI отвечает за текущую доступность моделей, биллинг, лимиты скорости и любые политики маршрутизации на стороне поставщика.

## Настройка

Установите Plugin, перезапустите Gateway, затем создайте API-ключ в GMI Cloud:

bashCopy code
[code]
    openclaw plugins install @openclaw/gmi-provideropenclaw gateway restart
[/code]

Затем выполните:

bashCopy code
[code]
    openclaw onboard --auth-choice gmi-api-key
[/code]

Или задайте:

bashCopy code
[code]
    export GMI_API_KEY="<your-gmi-api-key>" # pragma: allowlist secret
[/code]

## Значения по умолчанию

  * Поставщик: `gmi`
  * Псевдонимы: `gmi-cloud`, `gmicloud`
  * Базовый URL: `https://api.gmi-serving.com/v1`
  * Переменная среды: `GMI_API_KEY`
  * Модель по умолчанию: `gmi/google/gemini-3.1-flash-lite`


## Когда выбирать GMI

  * Вам нужна размещенная OpenAI-совместимая конечная точка, а не локальный сервер моделей.
  * Вы хотите попробовать несколько коммерческих семейств моделей и семейств с открытыми весами через одну учетную запись поставщика.
  * Вам нужен резервный поставщик с маршрутизацией upstream, отличающейся от OpenRouter, DeepInfra, Together или прямых API поставщиков.
  * Вам нужны специфичные для GMI идентификаторы моделей, цены или элементы управления учетной записью.


Выбирайте прямого поставщика вместо GMI, когда вам нужны нативные функции поставщика, которые GMI не предоставляет через свой OpenAI-совместимый маршрут. Выбирайте локального поставщика, такого как Ollama, LM Studio, vLLM или SGLang, когда локальность данных или локальное управление GPU важнее удобства размещенного сервиса.

## Модели

Каталог Plugin содержит начальные идентификаторы часто доступных маршрутов GMI Cloud, включая:

  * `gmi/zai-org/GLM-5.1-FP8`
  * `gmi/deepseek-ai/DeepSeek-V3.2`
  * `gmi/moonshotai/Kimi-K2.5`
  * `gmi/google/gemini-3.1-flash-lite`
  * `gmi/anthropic/claude-sonnet-4.6`
  * `gmi/openai/gpt-5.4`


Каталог является начальным списком, а не обещанием, что каждая учетная запись сможет вызывать каждую модель в любое время. Используйте команду OpenClaw для вывода списка моделей, чтобы увидеть, что настроенный поставщик сообщает в вашей среде:

bashCopy code
[code]
    openclaw models list --provider gmi
[/code]

## Устранение неполадок

  * `401` или `403`: проверьте, что `GMI_API_KEY` задана для процесса, в котором запущен OpenClaw, или повторно выполните onboarding, чтобы сохранить ключ в профиле аутентификации поставщика.
  * Ошибки неизвестной модели: убедитесь, что модель существует в вашей учетной записи GMI, и используйте полную ссылку `gmi/<route-id>`, показанную командой `openclaw models list --provider gmi`.
  * Периодические ошибки поставщика: попробуйте другой маршрут GMI или настройте GMI как резервного поставщика, а не единственного основного поставщика моделей.

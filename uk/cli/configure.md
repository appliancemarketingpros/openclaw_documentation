# `openclaw configure`

Інтерактивний запит для цільових змін наявного налаштування: облікові дані, пристрої, стандартні значення агента, Gateway, канали, плагіни, Skills і перевірки працездатності.

Використовуйте `openclaw onboard` для повного керованого першого запуску, `openclaw setup` лише для базової конфігурації/робочої області, а `openclaw channels add`, коли потрібно налаштувати тільки обліковий запис каналу.

Коли configure запускається з вибору автентифікації провайдера, засоби вибору стандартної моделі та списку дозволених автоматично віддають перевагу цьому провайдеру. Для пов’язаних провайдерів, як-от Volcengine і BytePlus, та сама перевага також відповідає їхнім варіантам плану кодування (`volcengine-plan/*`, `byteplus-plan/*`). Якщо фільтр бажаного провайдера дав би порожній список, configure повертається до нефільтрованого каталогу замість показу порожнього засобу вибору.

Для вебпошуку `openclaw configure --section web` дає змогу вибрати провайдера та налаштувати його облікові дані. Деякі провайдери також показують специфічні для провайдера додаткові запити:

  * **Grok** може запропонувати необов’язкове налаштування `x_search` з тим самим `XAI_API_KEY` і дати змогу вибрати модель `x_search`.
  * **Kimi** може запитати регіон Moonshot API (`api.moonshot.ai` чи `api.moonshot.cn`) і стандартну модель вебпошуку Kimi.


Пов’язане:

  * Довідка з конфігурації Gateway: [Конфігурація](</uk/gateway/configuration>)
  * Config CLI: [Config](</uk/cli/config>)


## Параметри

  * `--section <section>`: повторюваний фільтр розділів


Доступні розділи:

  * `workspace`
  * `model`
  * `web`
  * `gateway`
  * `daemon`
  * `channels`
  * `plugins`
  * `skills`
  * `health`


Примітки:

  * Вибір місця запуску Gateway завжди оновлює `gateway.mode`. Ви можете вибрати "Продовжити" без інших розділів, якщо це все, що вам потрібно.
  * Після запису локальної конфігурації configure встановлює вибрані завантажувані плагіни, коли вибраний шлях налаштування цього потребує. Конфігурація віддаленого Gateway не встановлює локальні пакети плагінів.
  * Сервіси, орієнтовані на канали (Slack/Discord/Matrix/Microsoft Teams), під час налаштування запитують списки дозволених каналів/кімнат. Ви можете вводити назви або ID; майстер за можливості перетворює назви на ID.
  * Якщо ви запускаєте крок встановлення демона, для автентифікації за токеном потрібен токен, а `gateway.auth.token` керується SecretRef, configure перевіряє SecretRef, але не зберігає розв’язані значення токена у відкритому тексті в метаданих середовища сервісу supervisor.
  * Якщо для автентифікації за токеном потрібен токен, а налаштований SecretRef токена не розв’язано, configure блокує встановлення демона з практичними рекомендаціями щодо виправлення.
  * Якщо налаштовано і `gateway.auth.token`, і `gateway.auth.password`, а `gateway.auth.mode` не задано, configure блокує встановлення демона, доки режим не буде задано явно.

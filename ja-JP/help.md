行き詰まりをすばやく解消するための、最も一般的な問題向けの経路:

  * [トラブルシューティング](</ja-JP/help/troubleshooting>) \- 症状起点の判断ツリー
  * [デバッグ](</ja-JP/help/debugging>) \- ウォッチモード、生ストリーム、開発プロファイル
  * [インストールの健全性確認](</ja-JP/install/node#troubleshooting>) \- Node / npm / PATH の確認
  * [Gateway のトラブルシューティング](</ja-JP/gateway/troubleshooting>) \- Gateway 固有の問題
  * [Doctor](</ja-JP/gateway/doctor>) \- 自動修復 + 診断バンドル


## よくある質問

  * [よくある質問](</ja-JP/help/faq>) \- 日常的な概念と運用上の質問
  * [初回実行のよくある質問](</ja-JP/help/faq-first-run>) \- インストール、オンボーディング、認証、サブスクリプション、初期の失敗
  * [モデルのよくある質問](</ja-JP/help/faq-models>) \- モデル選択、フェイルオーバー、認証プロファイル


## 診断

  * [環境変数](</ja-JP/help/environment>) \- OpenClaw が env vars を読み込む場所と優先順位
  * [診断フラグ](</ja-JP/diagnostics/flags>) \- ランタイム診断と詳細モード
  * [Node + tsx クラッシュ](</ja-JP/debug/node-issue>) \- 特定の Node / tsx ランタイムクラッシュシナリオ


## テスト

  * [テスト](</ja-JP/help/testing>) \- テストスイートと Docker ランナー
  * [更新と plugin テスト](</ja-JP/help/testing-updates-plugins>) \- パッケージ更新、移行、plugin インストール検証
  * [ライブテスト](</ja-JP/help/testing-live>) \- ネットワークに触れるプロバイダーと CLI のスモークテスト

## Вход вручную (рекомендуется)

Когда сайт требует входа, **войдите вручную** в профиле браузера **хоста** (браузере openclaw).

**Не** передавайте модели свои учетные данные. Автоматические входы часто срабатывают на антибот-защиту и могут заблокировать учетную запись.

Назад к основной документации по браузеру: [Браузер](</ru/tools/browser>).

## Какой профиль Chrome используется?

OpenClaw управляет **выделенным профилем Chrome** (с именем `openclaw`, интерфейс с оранжевым оттенком). Он отделен от вашего повседневного профиля браузера.

Для вызовов браузерного инструмента агентом:

  * Выбор по умолчанию: агент должен использовать свой изолированный браузер `openclaw`.
  * Используйте `profile="user"` только когда важны существующие сеансы с входом и пользователь находится за компьютером, чтобы нажать или подтвердить любой запрос на подключение.
  * Если у вас несколько пользовательских профилей браузера, укажите профиль явно, а не угадывайте.


Два простых способа открыть его:

  1. **Попросите агента открыть браузер** , а затем войдите самостоятельно.
  2. **Откройте его через CLI** :

bashCopy code
[code]
    openclaw browser startopenclaw browser open https://x.com
[/code]

Если у вас несколько профилей, передайте `--browser-profile <name>` (по умолчанию используется `openclaw`).

## X/Twitter: рекомендуемый процесс

  * **Чтение/поиск/треды:** используйте браузер **хоста** (вход вручную).
  * **Публикация обновлений:** используйте браузер **хоста** (вход вручную).


## Изоляция + доступ к браузеру хоста

Изолированные сеансы браузера **чаще** вызывают обнаружение ботов. Для X/Twitter (и других строгих сайтов) предпочитайте браузер **хоста**.

Если агент изолирован, браузерный инструмент по умолчанию использует изолированную среду. Чтобы разрешить управление хостом:

json5Copy code
[code]
    {  agents: {    defaults: {      sandbox: {        mode: "non-main",        browser: {          allowHostControl: true,        },      },    },  },}
[/code]

Затем откройте браузер хоста самостоятельно (вызовы CLI всегда выполняются для браузера хоста):

bashCopy code
[code]
    openclaw browser open https://x.com --browser-profile openclaw
[/code]

После установки `sandbox.browser.allowHostControl: true` вызовы инструмента `browser` агентом смогут обращаться к хосту. Либо отключите изоляцию для агента, который публикует обновления.

## การเข้าสู่ระบบด้วยตนเอง (แนะนำ)

เมื่อเว็บไซต์ต้องมีการเข้าสู่ระบบ ให้ **ลงชื่อเข้าใช้ด้วยตนเอง** ในโปรไฟล์เบราว์เซอร์ของ **โฮสต์** (เบราว์เซอร์ openclaw)

อย่าให้ข้อมูลรับรองของคุณแก่โมเดล การเข้าสู่ระบบอัตโนมัติมักกระตุ้นระบบป้องกันบอตและอาจทำให้บัญชีถูกล็อกได้

กลับไปยังเอกสารเบราว์เซอร์หลัก: [เบราว์เซอร์](</th/tools/browser>)

## ใช้โปรไฟล์ Chrome ใด?

OpenClaw ควบคุม **โปรไฟล์ Chrome เฉพาะ** (ชื่อ `openclaw`, UI โทนสีส้ม) โปรไฟล์นี้แยกจากโปรไฟล์เบราว์เซอร์ที่คุณใช้ประจำวัน

สำหรับการเรียกใช้เครื่องมือเบราว์เซอร์ของเอเจนต์:

  * ตัวเลือกเริ่มต้น: เอเจนต์ควรใช้เบราว์เซอร์ `openclaw` ที่แยกไว้ของตัวเอง
  * ใช้ `profile="user"` เฉพาะเมื่อเซสชันที่เข้าสู่ระบบไว้อยู่แล้วมีความสำคัญ และผู้ใช้อยู่ที่คอมพิวเตอร์เพื่อคลิก/อนุมัติพรอมป์แนบใดๆ
  * หากคุณมีโปรไฟล์เบราว์เซอร์ผู้ใช้หลายโปรไฟล์ ให้ระบุโปรไฟล์อย่างชัดเจนแทนการเดา


สองวิธีง่ายๆ ในการเข้าถึง:

  1. **ขอให้เอเจนต์เปิดเบราว์เซอร์** แล้วเข้าสู่ระบบด้วยตัวเอง
  2. **เปิดผ่าน CLI** :

bashCopy code
[code]
    openclaw browser startopenclaw browser open https://x.com
[/code]

หากคุณมีหลายโปรไฟล์ ให้ส่ง `--browser-profile <name>` (ค่าเริ่มต้นคือ `openclaw`)

## X/Twitter: ขั้นตอนที่แนะนำ

  * **อ่าน/ค้นหา/เธรด:** ใช้เบราว์เซอร์ของ **โฮสต์** (เข้าสู่ระบบด้วยตนเอง)
  * **โพสต์อัปเดต:** ใช้เบราว์เซอร์ของ **โฮสต์** (เข้าสู่ระบบด้วยตนเอง)


## Sandboxing + การเข้าถึงเบราว์เซอร์ของโฮสต์

เซสชันเบราว์เซอร์แบบ Sandbox มีแนวโน้ม **มากกว่า** ที่จะกระตุ้นการตรวจจับบอต สำหรับ X/Twitter (และเว็บไซต์อื่นๆ ที่เข้มงวด) ให้เลือกใช้เบราว์เซอร์ของ **โฮสต์**

หากเอเจนต์อยู่ใน Sandbox เครื่องมือเบราว์เซอร์จะใช้ Sandbox เป็นค่าเริ่มต้น เมื่อต้องการอนุญาตให้ควบคุมโฮสต์:

json5Copy code
[code]
    {  agents: {    defaults: {      sandbox: {        mode: "non-main",        browser: {          allowHostControl: true,        },      },    },  },}
[/code]

จากนั้นเปิดเบราว์เซอร์ของโฮสต์ด้วยตัวเอง (การเรียกใช้ CLI จะทำงานกับเบราว์เซอร์ของโฮสต์เสมอ):

bashCopy code
[code]
    openclaw browser open https://x.com --browser-profile openclaw
[/code]

จากนั้นการเรียกใช้เครื่องมือ `browser` ของเอเจนต์จะสามารถกำหนดเป้าหมายไปที่โฮสต์ได้เมื่อมีการตั้งค่า `sandbox.browser.allowHostControl: true` แล้ว อีกทางหนึ่งคือปิดใช้ Sandbox สำหรับเอเจนต์ที่โพสต์อัปเดต

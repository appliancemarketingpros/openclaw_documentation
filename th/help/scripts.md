ไดเรกทอรี `scripts/` มีสคริปต์ช่วยสำหรับเวิร์กโฟลว์ภายในเครื่องและงานปฏิบัติการ ใช้สคริปต์เหล่านี้เมื่องานเชื่อมโยงกับสคริปต์อย่างชัดเจน มิฉะนั้นให้เลือกใช้ CLI

## ข้อตกลง

  * สคริปต์เป็น **ทางเลือก** เว้นแต่จะถูกอ้างอิงในเอกสารหรือเช็กลิสต์การเผยแพร่
  * เลือกใช้พื้นผิว CLI เมื่อมีอยู่ (ตัวอย่าง: การตรวจสอบการยืนยันตัวตนใช้ `openclaw models status --check`)
  * ถือว่าสคริปต์เฉพาะเจาะจงกับโฮสต์ ให้อ่านสคริปต์ก่อนรันบนเครื่องใหม่


## สคริปต์ตรวจสอบการยืนยันตัวตน

การตรวจสอบการยืนยันตัวตนครอบคลุมอยู่ใน [การยืนยันตัวตน](</th/gateway/authentication>) สคริปต์ภายใต้ `scripts/` เป็นส่วนเสริมทางเลือกสำหรับเวิร์กโฟลว์โทรศัพท์ systemd/Termux

## ตัวช่วยอ่าน GitHub

ใช้ `scripts/gh-read` เมื่อคุณต้องการให้ `gh` ใช้โทเค็นการติดตั้ง GitHub App สำหรับการเรียกอ่านที่มีขอบเขตเฉพาะรีโป โดยยังคงให้ `gh` ปกติใช้การเข้าสู่ระบบส่วนตัวของคุณสำหรับการดำเนินการเขียน

env ที่จำเป็น:

  * `OPENCLAW_GH_READ_APP_ID`
  * `OPENCLAW_GH_READ_PRIVATE_KEY_FILE`


env ทางเลือก:

  * `OPENCLAW_GH_READ_INSTALLATION_ID` เมื่อคุณต้องการข้ามการค้นหาการติดตั้งตามรีโป
  * `OPENCLAW_GH_READ_PERMISSIONS` เป็นการแทนที่แบบคั่นด้วยจุลภาคสำหรับชุดย่อยสิทธิ์การอ่านที่จะร้องขอ


ลำดับการระบุรีโป:

  * `gh ... -R owner/repo`
  * `GH_REPO`
  * `git remote origin`


ตัวอย่าง:

  * `scripts/gh-read pr view 123`
  * `scripts/gh-read run list -R openclaw/openclaw`
  * `scripts/gh-read api repos/openclaw/openclaw/pulls/123`


## เมื่อเพิ่มสคริปต์

  * รักษาให้สคริปต์มีขอบเขตชัดเจนและมีเอกสารกำกับ
  * เพิ่มรายการสั้น ๆ ในเอกสารที่เกี่ยวข้อง (หรือสร้างใหม่หากยังไม่มี)

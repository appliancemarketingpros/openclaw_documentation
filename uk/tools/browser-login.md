## Ручний вхід (рекомендовано)

Коли сайт вимагає входу, **увійдіть вручну** у профілі браузера **хоста** (браузер openclaw).

**Не** передавайте моделі свої облікові дані. Автоматизовані входи часто запускають антибот-захист і можуть заблокувати обліковий запис.

Назад до основної документації браузера: [Браузер](</uk/tools/browser>).

## Який профіль Chrome використовується?

OpenClaw керує **виділеним профілем Chrome** (з назвою `openclaw`, інтерфейс із помаранчевим відтінком). Він окремий від вашого щоденного профілю браузера.

Для викликів браузерного інструмента агентом:

  * Вибір за замовчуванням: агент має використовувати свій ізольований браузер `openclaw`.
  * Використовуйте `profile="user"` лише тоді, коли важливі наявні сесії з входом, а користувач перебуває за комп’ютером, щоб натиснути або підтвердити будь-який запит на підключення.
  * Якщо у вас кілька профілів користувацького браузера, явно вкажіть профіль замість припущень.


Два прості способи отримати до нього доступ:

  1. **Попросіть агента відкрити браузер** , а потім увійдіть самостійно.
  2. **Відкрийте його через CLI** :

bashCopy code
[code]
    openclaw browser startopenclaw browser open https://x.com
[/code]

Якщо у вас кілька профілів, передайте `--browser-profile <name>` (за замовчуванням використовується `openclaw`).

## X/Twitter: рекомендований процес

  * **Читання/пошук/гілки:** використовуйте браузер **хоста** (ручний вхід).
  * **Публікація оновлень:** використовуйте браузер **хоста** (ручний вхід).


## Пісочниця + доступ до браузера хоста

Сеанси браузера в пісочниці **частіше** запускають виявлення ботів. Для X/Twitter (та інших суворих сайтів) віддавайте перевагу браузеру **хоста**.

Якщо агент працює в пісочниці, браузерний інструмент за замовчуванням використовує пісочницю. Щоб дозволити керування хостом:

json5Copy code
[code]
    {  agents: {    defaults: {      sandbox: {        mode: "non-main",        browser: {          allowHostControl: true,        },      },    },  },}
[/code]

Потім самостійно відкрийте браузер хоста (виклики CLI завжди виконуються щодо браузера хоста):

bashCopy code
[code]
    openclaw browser open https://x.com --browser-profile openclaw
[/code]

Після встановлення `sandbox.browser.allowHostControl: true` виклики інструмента `browser` агентом зможуть націлюватися на хост. Або вимкніть пісочницю для агента, який публікує оновлення.

ModelsProviders

GMI Cloud یک پلتفرم استنتاج میزبانی‌شده برای مدل‌های پیشرو و مدل‌های با وزن‌های باز است که پشت یک API سازگار با OpenAI قرار دارد. در OpenClaw، این یک Plugin رسمیِ ارائه‌دهندهٔ خارجی است؛ یعنی آن را یک‌بار نصب می‌کنید، با شناسهٔ ارائه‌دهندهٔ `gmi` انتخابش می‌کنید، اعتبارنامه‌ها را از طریق احراز هویت عادی مدل ذخیره می‌کنید و از ارجاع‌های مدل مانند `gmi/google/gemini-3.1-flash-lite` استفاده می‌کنید.

وقتی GMI را به کار ببرید که برای چند خانوادهٔ مدل میزبانی‌شده، از جمله مسیرهای Google، Anthropic، OpenAI، DeepSeek، Moonshot و Z.AI که در کاتالوگ GMI عرضه شده‌اند، یک کلید API واحد می‌خواهید. این گزینه به‌عنوان ارائه‌دهندهٔ ثانویه برای جایگزینی مدل، برای مقایسهٔ مسیرهای میزبانی‌شده میان فروشندگان، یا زمانی که GMI مدلی را پیش از ارائه‌دهندهٔ اصلی شما در دسترس دارد، مفید است.

این ارائه‌دهنده از معناشناسی چت سازگار با OpenAI استفاده می‌کند. OpenClaw مالک شناسهٔ ارائه‌دهنده، پروفایل احراز هویت، نام‌های مستعار، بذر کاتالوگ مدل و URL پایه است؛ GMI مالک دسترس‌پذیری زندهٔ مدل‌ها، صورت‌حساب، محدودیت‌های نرخ و هر سیاست مسیریابی سمت ارائه‌دهنده است.

## راه‌اندازی

Plugin را نصب کنید، gateway را دوباره راه‌اندازی کنید، سپس در GMI Cloud یک کلید API بسازید:

bashCopy code
[code]
    openclaw plugins install @openclaw/gmi-provideropenclaw gateway restart
[/code]

سپس اجرا کنید:

bashCopy code
[code]
    openclaw onboard --auth-choice gmi-api-key
[/code]

یا تنظیم کنید:

bashCopy code
[code]
    export GMI_API_KEY="<your-gmi-api-key>" # pragma: allowlist secret
[/code]

## پیش‌فرض‌ها

  * ارائه‌دهنده: `gmi`
  * نام‌های مستعار: `gmi-cloud`, `gmicloud`
  * URL پایه: `https://api.gmi-serving.com/v1`
  * متغیر محیطی: `GMI_API_KEY`
  * مدل پیش‌فرض: `gmi/google/gemini-3.1-flash-lite`


## چه زمانی GMI را انتخاب کنید

  * به‌جای سرور مدل محلی، یک نقطهٔ پایانی میزبانی‌شدهٔ سازگار با OpenAI می‌خواهید.
  * می‌خواهید چند خانوادهٔ مدل تجاری و با وزن‌های باز را از طریق یک حساب ارائه‌دهنده امتحان کنید.
  * ارائه‌دهندهٔ جایگزینی می‌خواهید که مسیریابی بالادستی متفاوتی نسبت به OpenRouter، DeepInfra، Together یا APIهای مستقیم فروشندگان داشته باشد.
  * به شناسه‌های مدل، قیمت‌گذاری یا کنترل‌های حساب مخصوص GMI نیاز دارید.


وقتی به قابلیت‌های بومی فروشنده نیاز دارید که GMI از طریق مسیر سازگار با OpenAI خود عرضه نمی‌کند، به‌جای آن ارائه‌دهندهٔ مستقیم فروشنده را انتخاب کنید. وقتی محل‌مندی داده یا کنترل GPU محلی از راحتی میزبانی‌شده مهم‌تر است، ارائه‌دهندهٔ محلی مانند Ollama، LM Studio، vLLM یا SGLang را انتخاب کنید.

## مدل‌ها

کاتالوگ Plugin شناسه‌های مسیر رایج در دسترس GMI Cloud را بذرگذاری می‌کند، از جمله:

  * `gmi/zai-org/GLM-5.1-FP8`
  * `gmi/deepseek-ai/DeepSeek-V3.2`
  * `gmi/moonshotai/Kimi-K2.5`
  * `gmi/google/gemini-3.1-flash-lite`
  * `gmi/anthropic/claude-sonnet-4.6`
  * `gmi/openai/gpt-5.4`


کاتالوگ یک بذر است، نه تضمینی که هر حساب بتواند همیشه هر مدل را فراخوانی کند. برای دیدن آنچه ارائه‌دهندهٔ پیکربندی‌شده در محیط شما گزارش می‌کند، از فرمان فهرست‌گیری مدل‌های OpenClaw استفاده کنید:

bashCopy code
[code]
    openclaw models list --provider gmi
[/code]

## عیب‌یابی

  * `401` یا `403`: بررسی کنید که `GMI_API_KEY` برای فرایندی که OpenClaw را اجرا می‌کند تنظیم شده باشد، یا راه‌اندازی اولیه را دوباره اجرا کنید تا کلید در پروفایل احراز هویت ارائه‌دهنده ذخیره شود.
  * خطاهای مدل ناشناخته: تأیید کنید مدل در حساب GMI شما وجود دارد و از ارجاع کامل `gmi/<route-id>` که با `openclaw models list --provider gmi` نشان داده می‌شود استفاده کنید.
  * خطاهای متناوب ارائه‌دهنده: یک مسیر GMI دیگر را امتحان کنید یا GMI را به‌جای تنها ارائه‌دهندهٔ مدل اصلی، به‌عنوان جایگزین پیکربندی کنید.

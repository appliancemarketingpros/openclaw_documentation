# 成熟度評分卡

發行就緒度 - 由分類法 + QA 證據產生

實用檢視哪些項目已就緒、哪些已有證據支持，以及哪些仍需改進。

50 個介面 - 281 個能力領域 - 決定性覆蓋率，加上經人工審查的品質與完整性。

瀏覽介面 / 檢查 QA 證據 / [閱讀分類法](</zh-TW/maturity/taxonomy>)

## 此頁面的用途

使用此頁面回答一個問題：哪些 OpenClaw 介面是發行時可信賴的選擇，而哪些證據支持這項判斷？覆蓋率來自決定性的 QA 證據；品質與完整性則以經審查的成熟度分數維護。

## 概覽

67% 成熟度分數

Alpha 版 品質 + 完整性 覆蓋率：實驗性 - 4% 品質：Alpha 版 - 63% 完整性：Beta 版 - 70%

覆蓋率刻意由證據主導：一個領域不會只因為實作已存在就變成「已就緒」。它不是成熟度分數的輸入，但 OpenClaw 的目標是隨著時間，讓成熟且達穩定版或更高等級功能的端對端覆蓋率維持在 90% 以上。

## 分數區間

實驗性0-50%

Alpha 版50-70%

Beta 版70-80%

穩定版80-95%

Clawesome95-100%
